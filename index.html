求两个数的和，不允许使用加法和减法

当看到这个题目的时候，大概就有两种方法来实现；
一种是只要是和算术相关的操作都可以使用位操作来进行变相实现。
最先想到的就是位操作来解决，如下： 

int add1(int x, int y) 
{
    if (y == 0) {
        return x;
    }

    return add1(x ^ y, (x & y) << 1);

}

把加法拆成两部分来操作，一部分是做简单的加法，另一部分是做进位操作； 其中x^y是简单的加，无进位。x&y是求进位，(x & y) << 1表示进位的数左移一 位和之前无进位的数做求和；

另一种是利用某种机制来做间接的调用加减法,利用a[i] 等价于*(a+i)机制：

long add2(int x, int y)
{
    return (long)(&((char*)x)[y]);
}

首先我们知道，a[i]是和(a+i)是等价的然而加i会偏移多少，这个取决于a的类型，如果a 是char型的则加1就是加1字节；这个地方就利用这个机制，(char)x把整数x强转为char类型的 指针想当与数组名首地址a，然后((char)x)[y]这个就等价于（x+y）取对应空间里面的值， 这个时候我们并不想要这个地址里存储的值，而是想要这个值对应的地址，所以&((char)x)[y] 进行和符合抵消，最后把这个地址转换为long类型； 注意：指针在64位操作系统里面是8字节。
